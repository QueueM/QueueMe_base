Introduction to QueueMe API

The QueueMe API is a RESTful API that allows you to integrate queue management and booking functionality into your applications. With our API, you can create and manage queues, services, bookings, specialists, and more.

This guide provides an overview of the API functionality and how to use it effectively. For detailed endpoint documentation, please refer to our Swagger UI or ReDoc interfaces.

Base URL

All API requests should be made to:

Base URL 
https://api.queueme.net/api/v1/

API Versioning

The QueueMe API uses URL versioning. The current version is v1. When a new version is released, the old version will remain available for a deprecation period before being removed.

Authentication

The QueueMe API uses JSON Web Tokens (JWT) for authentication. To authenticate your requests, you need to include an Authorization header with a Bearer token.

Obtaining a Token

To obtain a token, make a POST request to the authentication endpoint with your credentials:

cURL 
curl -X POST https://api.queueme.net/api/v1/auth/token/ \
     -H "Content-Type: application/json" \
     -d '{
       "email": "your.email@example.com",
       "password": "your_password"
     }'

The response will include an access_token and a refresh_token:

JSON Response 
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600
}

Using the Token

Include the access token in the Authorization header of your requests:

cURL 
curl -X GET "https://api.queueme.net/api/v1/shops/" \
     -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
     -H "Content-Type: application/json"

Refreshing Tokens

Access tokens expire after a certain period. Use the refresh token to obtain a new access token:

cURL 
curl -X POST https://api.queueme.net/api/v1/auth/token/refresh/ \
     -H "Content-Type: application/json" \
     -d '{
       "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
     }'
Security Note
Keep your refresh tokens secure and never expose them to client-side code. Refresh tokens should be stored securely on your backend servers.

Making Requests

The QueueMe API accepts requests with JSON payloads and returns JSON responses. All endpoints require the Content-Type: application/json header.

HTTP Methods

The API uses standard HTTP methods to perform different operations:

Method Description
GET Retrieve resources
POST Create resources
PUT Update resources (complete replacement)
PATCH Update resources (partial update)
DELETE Delete resources

Request Headers

Common headers used in API requests:

Header Description
Authorization Authentication token in format: Bearer <token>
Content-Type Media type of the request body, usually application/json
Accept Media type(s) accepted for the response, usually application/json
Accept-Language Preferred language for responses (e.g., en-US, ar)

Responses

All API responses are returned in JSON format with appropriate HTTP status codes.

Response Structure

Successful responses typically include a data object containing the requested information:

JSON Response 
{
  "id": 123,
  "name": "Example Shop",
  "address": "123 Main St",
  "phone": "+1234567890",
  "email": "info@example.com",
  "created_at": "2025-06-15T10:30:45Z",
  "updated_at": "2025-06-15T10:30:45Z"
}

Response Status Codes

The API uses standard HTTP status codes to indicate the success or failure of a request:

Status Code Description
200 OK The request was successful
201 Created A new resource was successfully created
204 No Content The request was successful but no content is returned
400 Bad Request The request was invalid or could not be understood
401 Unauthorized Authentication failed or credentials were not provided
403 Forbidden The authenticated user doesn't have permission to access the resource
404 Not Found The requested resource was not found
422 Unprocessable Entity The request was well-formed but had semantic errors
429 Too Many Requests Rate limit exceeded
500 Internal Server Error An error occurred on the server

Error Handling

When an error occurs, the API returns an error response with details about what went wrong.

Error Response Format

Error responses follow a consistent format:

JSON Error Response 
{
  "error": {
    "code": "validation_error",
    "message": "Invalid input provided",
    "details": {
      "name": ["This field is required"],
      "email": ["Enter a valid email address"]
    }
  }
}
Tip
Always check for error responses in your client application and handle them appropriately. Display user-friendly error messages instead of raw API errors.

Pagination

List endpoints return paginated results to improve performance and reduce response size. Pagination information is included in the response metadata.

Pagination Parameters

You can control pagination using the following query parameters:

Parameter Description Default
page Page number to retrieve 1
limit Number of items per page 20

Pagination Response

Paginated responses include metadata about the current page and total results:

JSON Paginated Response 
{
  "results": [
    {
      "id": 1,
      "name": "Shop 1",
      ...
    },
    {
      "id": 2,
      "name": "Shop 2",
      ...
    }
  ],
  "meta": {
    "page": 1,
    "limit": 20,
    "total_items": 150,
    "total_pages": 8,
    "next": "/api/v1/shops/?page=2&limit=20",
    "previous": null
  }
}

API Examples

Here are some common examples of using the QueueMe API.

Creating a Booking

This example shows how to create a new booking for a service:

cURL 
curl -X POST "https://api.queueme.net/api/v1/bookings/" \
     -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
     -H "Content-Type: application/json" \
     -d '{
       "service_id": 123,
       "specialist_id": 456,
       "date": "2025-06-15",
       "time_slot": "14:30",
       "customer_id": 789,
       "notes": "First-time customer"
     }'

Getting Available Time Slots

Retrieve available time slots for a specific service and date:

cURL 
curl -X GET "https://api.queueme.net/api/v1/services/123/slots?date=2025-06-15" \
     -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
     -H "Content-Type: application/json"

Updating a Queue Item Status

Update the status of a queue item:

cURL 
curl -X PATCH "https://api.queueme.net/api/v1/queues/items/123/" \
     -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
     -H "Content-Type: application/json" \
     -d '{
       "status": "serving"
     }'
