Introduction

The QueueMe API provides programmatic access to the QueueMe platform. It allows developers to integrate QueueMe's queue management, booking, and specialist management features into their applications.

Base URL

https://api.queueme.net/api

Response Format

All API responses are returned in JSON format.

Standard response structure:

{
  "data": { /* Response data */ },
  "meta": {
    "pagination": {
      "total": 100,
      "count": 20,
      "per_page": 20,
      "current_page": 1,
      "total_pages": 5
    }
  }
}

Authentication

QueueMe API uses JWT (JSON Web Tokens) for authentication. To access protected endpoints, you'll need to:

  1. Obtain a JWT token using your credentials
  2. Include the token in the Authorization header of your requests
POST /api/auth/token/

Obtain a JWT token by providing your phone number and password.

Request
Response
{
  "phone": "966558173151",
  "password": "your_password"
}
{
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Using the Token

For all authenticated requests, include the access token in your Authorization header:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Token Refresh

POST /api/auth/token/refresh/

Refresh your access token using a valid refresh token.

Error Handling

The QueueMe API uses conventional HTTP response codes to indicate the success or failure of an API request. In general:

  • 2xx status codes indicate success
  • 4xx status codes indicate an error in the request
  • 5xx status codes indicate an error on the server

Error Response Format

{
  "error": {
    "code": "invalid_credentials",
    "message": "Unable to login with provided credentials.",
    "details": { /* Additional error details */ }
  }
}

Common Error Codes

Status Code Description
400 Bad Request The request was malformed or contained invalid parameters
401 Unauthorized Authentication credentials were missing or invalid
403 Forbidden The authenticated user doesn't have permission for the requested resource
404 Not Found The requested resource does not exist
429 Too Many Requests Rate limit exceeded
500 Internal Server Error An error occurred on the server

Pagination

List endpoints in the QueueMe API support pagination. By default, requests return 20 items per page.

Pagination Parameters

Parameter Description
page Page number (starts at 1)
page_size Number of items per page (default: 20, max: 100)

Pagination Response

Paginated responses include metadata in the meta.pagination object: 

{
  "data": [ /* Response data */ ],
  "meta": {
    "pagination": {
      "total": 100,
      "count": 20,
      "per_page": 20,
      "current_page": 1,
      "total_pages": 5
    }
  }
}

Authentication API

The Authentication API allows users to register, login, and manage their account.

POST /api/auth/register/

Register a new user account.

Parameters

Name Type Required Description
phone string Yes Phone number
password string Yes Password
password_confirm string Yes Password confirmation
first_name string Yes First name
last_name string Yes Last name
email string No Email address
POST /api/auth/verify-otp/

Verify OTP for phone verification.

Parameters

Name Type Required Description
phone string Yes Phone number
otp string Yes OTP code

Shops API

The Shops API allows you to manage and retrieve information about shops on the QueueMe platform.

GET /api/shops/

List all shops with pagination support.

Query Parameters

Parameter Type Required Description
page integer No Page number
page_size integer No Results per page (default: 20, max: 100)
search string No Search by shop name
city string No Filter by city
category integer No Filter by category ID
is_active boolean No Filter by active status
